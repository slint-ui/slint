`global Name { /* .. properties or callbacks .. */ }` declares a global singleton
whose properties and callbacks are available throughout the project.
They are accessed as `Name.property`:

```slint
global Palette  {
    in-out property<color> primary: blue;
    in-out property<color> secondary: green;
}

export component Example inherits Rectangle {
    background: Palette.primary;
    border-color: Palette.secondary;
    border-width: 2px;
}
```

## Exported Globals

An [exported](../exports/) global is accessible from other files.
A global re-exported from the file that also exports the main application component
is visible to the native code of the application:

```slint
export global Logic  {
    in-out property <int> the-value;
    pure callback magic-operation(int) -> int;
}
// ...
```

:::note[Note]
Global singletons aren't shared between separate windows, or between a window and a `SystemTrayIcon` exported alongside it.
The global callbacks and properties may need to be initialized for each instance.
:::

## Re-Exposing Global Properties

A [two-way binding](../two-way-bindings/) re-exposes a callback or property of a global on a component:

```slint
global Logic  {
    in-out property <int> the-value;
    pure callback magic-operation(int) -> int;
}

component SomeComponent inherits Text {
    // use the global in any component
    text: "The magic value is:" + Logic.magic-operation(42);
}

export component MainWindow inherits Window {
    // re-expose the global properties such that the native code
    // can access or modify them
    in-out property the-value <=> Logic.the-value;
    pure callback magic-operation <=> Logic.magic-operation;

    SomeComponent {}
}
```
