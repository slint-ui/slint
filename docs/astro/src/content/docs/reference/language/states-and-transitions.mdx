## States

The `states` statement declares states and sets properties of multiple elements in one go:

```slint
export component Example inherits Window {
    preferred-width: 100px;
    preferred-height: 100px;
    default-font-size: 24px;

    label := Text { }
    ta := TouchArea {
        clicked => {
            active = !active;
        }
    }
    property <bool> active: true;
    states [
        active when active && !ta.has-hover: {
            label.text: "Active";
            root.background: blue;
        }
        active-hover when active && ta.has-hover: {
            label.text: "Active\nHover";
            root.background: green;
        }
        inactive when !active: {
            label.text: "Inactive";
            root.background: gray;
        }
    ]
}
```

Each state has a name, a `when` condition, and a set of property values that apply while the condition holds.
In this example, hovering toggles between a blue and a green background and adjusts the label,
and clicking enters the `inactive` state.

## Transitions

Transitions bind [animations](../animations/) to state changes.

This example defines two transitions.
The `out` keyword animates all properties for 800ms when leaving the `disabled` state.
The `in` keyword animates the background when transitioning into the `down` state:

```slint
export component Example inherits Window {
    preferred-width: 100px;
    preferred-height: 100px;

    text := Text { text: "hello"; }
    in-out property<bool> pressed;
    in-out property<bool> is-enabled;

    states [
        disabled when !root.is-enabled : {
            background: gray; // same as root.background: gray;
            text.color: white;
            out {
                animate * { duration: 800ms; }
            }
        }
        down when pressed : {
            background: blue;
            in {
                animate background { duration: 300ms; }
            }
        }
    ]
}
```

There are three types of transitions:

- **`in`**: animates properties when entering a state.
- **`out`**: animates properties when leaving a state.
- **`in-out`**: animates properties both when entering and leaving a state.

The `in-out` transition plays the same animation for entering and exiting a state,
avoiding a duplicated animation definition.
