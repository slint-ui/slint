Use a `RadioGroup` to let the user select exactly one option from a mutually exclusive list. The `options` property takes an array of [`RadioOption`](#radiooption).

## RadioOption

`RadioOption` defines one entry in a `RadioGroup`.

- **`label`** (`string`) - text shown next to the radio button.
- **`value`** (`string`) - stable programmatic id used for the group `value` and in `changed(value)`.
- **`enabled`** (`bool`) - when `false`, this specific option is visible but not selectable.

```
{ label: "Desktop", value: "desktop", enabled: true }
```

## Properties

### enabled
<SlintProperty propName="enabled" typeName="bool" defaultValue="true">
When `false`, the entire group and all its radio buttons cannot be interacted with.
</SlintProperty>

### has-focus
<SlintProperty propName="has-focus" typeName="bool" defaultValue="false" propertyVisibility="out">
Set to `true` when any radio button inside the group has keyboard focus.
</SlintProperty>

### options
<SlintProperty propName="options" typeName="[struct]" structName="RadioOption" defaultValue="[]">
The list of options to display. Each entry has a `label`, a stable `value`, and an `enabled` flag.

```slint 'options: [{ label: "A", value: "a", enabled: true }];'
RadioGroup {
    options: [
        { label: "Desktop", value: "desktop", enabled: true },
        { label: "Mobile", value: "mobile", enabled: true },
        { label: "Embedded", value: "embedded", enabled: true },
    ];
}
```
</SlintProperty>

### title
<SlintProperty propName="title" typeName="string" defaultValue='""'>
An optional label rendered above the group. When empty, no title is shown.

```slint 'title: "Target Platform";'
RadioGroup {
    title: "Target Platform";
    options: [
        { label: "Desktop", value: "desktop", enabled: true },
        { label: "Mobile", value: "mobile", enabled: true },
        { label: "Embedded", value: "embedded", enabled: true },
    ];
}
```
</SlintProperty>

### value
<SlintProperty propName="value" typeName="string" defaultValue='""' propertyVisibility="in-out">
The `value` field of the currently selected `RadioOption`. Setting this property programmatically updates which radio button appears checked. Each option's `value` must be unique within the group.

```slint 'value: "desktop";'
RadioGroup {
    options: [
        { label: "Desktop", value: "desktop", enabled: true },
        { label: "Mobile", value: "mobile", enabled: true },
        { label: "Embedded", value: "embedded", enabled: true },
    ];
    value: "desktop";
}
```
</SlintProperty>

### vertical
<SlintProperty propName="vertical" typeName="bool" defaultValue="true">
When `true` (default), radio buttons are stacked vertically. Set to `false` to arrange them horizontally.

```slint 'vertical: false;'
RadioGroup {
    vertical: false;
    options: [
        { label: "Desktop", value: "desktop", enabled: true },
        { label: "Mobile", value: "mobile", enabled: true },
        { label: "Embedded", value: "embedded", enabled: true },
    ];
}
```
</SlintProperty>

## Callbacks

### changed(string)
Invoked when the user selects a different option. The argument is the `value` of the newly selected `RadioOption`.

```slint {7-9}
RadioGroup {
    options: [
        { label: "Desktop", value: "desktop", enabled: true },
        { label: "Mobile", value: "mobile", enabled: true },
        { label: "Embedded", value: "embedded", enabled: true },
    ];
    changed(value) => {
        debug("Selected platform: ", value);
    }
}
```
