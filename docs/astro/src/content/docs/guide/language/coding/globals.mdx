---
title: Globals
description: Sharing state across your whole project with global singletons.
---

Declare a global singleton with `global Name { /* .. properties or callbacks .. */ }` to
make properties and callbacks available throughout the entire project.
Access them using `Name.property`.

For example, this can be useful for a common color palette:

```slint
global Palette  {
    in-out property<color> primary: blue;
    in-out property<color> secondary: green;
}

export component Example inherits Rectangle {
    background: Palette.primary;
    border-color: Palette.secondary;
    border-width: 2px;
}
```

Export a global to make it accessible from other files.
A global re-exported from the file that also exports your main application component
becomes visible to the business logic, so you can set its properties and
implement its callbacks from Rust, C++, JavaScript, or Python:

```slint
export global Logic  {
    in-out property <int> the-value;
    pure callback magic-operation(int) -> int;
}
// ...
```

The [Globals](../../../../reference/language/globals/) chapter of the language reference
shows the native-code side for each language, and how to re-expose global properties
on a component with two-way bindings.

:::note[Note]
Global singletons aren't shared between separate windows.
You may need to initialize the global callbacks and properties for each instance you create.
:::
