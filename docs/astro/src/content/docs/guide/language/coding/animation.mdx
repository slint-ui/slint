Declare animations for properties with the `animate` keyword like this:

```slint
export component Example inherits Window {
    preferred-width: 100px;
    preferred-height: 100px;

    background: area.pressed ? blue : red;
    animate background {
        duration: 250ms;
    }

    area := TouchArea {}
}
```

This will animate the color property for 250ms whenever it changes.

It's also possible to animate several properties with the same animation, so:

```slint no-test
animate x, y { duration: 100ms; easing: ease-out-bounce; }
```

is the same as:

```slint no-test
animate x { duration: 100ms; easing: ease-out-bounce; }
animate y { duration: 100ms; easing: ease-out-bounce; }
```


Fine-tune animations using the following parameters:

### delay
<SlintProperty propName="delay" typeName="duration">
The amount of time to wait before starting the animation.
</SlintProperty>

### duration

<SlintProperty propName="duration" typeName="duration">
 The amount of time it takes for the animation to complete.
</SlintProperty>

### iteration-count

<SlintProperty propName="iteration-count" typeName="float">
The number of times an animation should run. A negative value specifies
infinite reruns. Fractional values are possible.
For permanently running animations, see <Link type="AnimationTick" label="`animation-tick()`" />.
</SlintProperty>

### easing

<SlintProperty propName="easing" typeName="easing">
Can be any of the following. See [`easings.net`](https://easings.net/) for a visual reference: <Link type="easing" label="Easings"/>
</SlintProperty>

### direction
<SlintProperty propName="direction" typeName="enum" enumName="AnimationDirection">
Use this to set or change the direction of the animation.
</SlintProperty>

### enabled
<SlintProperty propName="enabled" typeName="bool" defaultValue="true">
Controls whether the animation runs. When set to `false`, the property value changes immediately to its target without delay, easing, or iterations.
</SlintProperty>

## Spring Animation Types

Two ways to define spring animations exist and the animation will fail to compile if properties from both types are used at the same time.

The animation is considered settled (finished) once it has a low velocity and it is very close to the final position.

### 1. `bounce` and `duration`
This version provides an intuitive sense for how the spring animation will work and defaults to a critically damped spring (`bounce` = 0).

Since `duration` helps define the spring's [natural frequency](https://en.wikipedia.org/wiki/Natural_frequency), the animation is not guaranteed to be done after `duration` elapses. If `bounce` is close to 0, the animation will be nearly done, however the further from 0 `bounce` is, the longer the animation will take. If `bounce` is 1, the animation will never finish.

#### Bounce values
- `bounce` = 0: Value will approach target as quickly as possible without going past
- `bounce` > 0: Value will approach target quickly but will overshoot
- `bounce` < 0: Value will approach target slowly

### 2. `mass`, `stiffness`, and `damping`
This version provides fine control on how exactly the spring will work, and is advised against unless needed. This does not respect `duration` and will stop once it is settled.

### Details
The spring's animation relies on the [natural frequency](https://en.wikipedia.org/wiki/Natural_frequency)(ω₀) and the [damping ratio](https://en.wikipedia.org/wiki/Damping#Damping_ratio)(ζ). These values are derived from `bounce`/`duration` or `mass`/`stiffness`/`damping` to feed the [mass-spring-damper model](https://en.wikipedia.org/wiki/Mass-spring-damper_model)
