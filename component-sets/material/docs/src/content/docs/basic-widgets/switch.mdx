Use a `Switch` to let the user toggle between on and off states, for example to enable or disable a feature. Consider using a `CheckBox` element instead if the action resembles more a selection of options.

## Properties

### checked
<SlintProperty propName="checked" typeName="bool" defaultValue="false" propertyVisibility="in-out">
Whether the switch is checked (on) or not (off).
```slint "checked: true;"
Switch {
    checked: true;
}
```
</SlintProperty>

### enabled
<SlintProperty propName="enabled" typeName="bool" defaultValue="true">
Defaults to true. When false, the switch can't be toggled.
</SlintProperty>

### tooltip
<SlintProperty propName="tooltip" typeName="string">
The tooltip text for the switch.
```slint 'tooltip: "Enable feature";'
Switch {
    tooltip: "Toggle me";
}
```
</SlintProperty>

### on_icon
<SlintProperty propName="on_icon" typeName="image">
The icon to show when the switch is checked (on).
```slint 'on_icon: @image-url("icon_on.png");'
Switch {
    on_icon: @image-url("icon_on.png");
}
```
</SlintProperty>

### off_icon
<SlintProperty propName="off_icon" typeName="image">
The icon to show when the switch is not checked (off).
```slint 'off_icon: @image-url("icon_off.png");'
Switch {
    off_icon: @image-url("icon_off.png");
}
```
</SlintProperty>

## Callbacks

### toggled()
The switch value changed

```slint {3-5}
Switch {
    changed checked => {
        debug("Switch checked: ", self.checked);
    }
}
```
